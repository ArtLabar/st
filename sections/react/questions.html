Чем отличаются stateful-компоненты от stateless-компонентов?

Первые имеют состояние (классовые компоненты) в отличии от вторых (функциональных).

Если задача компонента только в отрисовке UI, какого вида компонент лучше использовать и почему?

Функциональный, так как он проще в понимании и тестировании.

Влияет ли на производительность выбор вида компонента - классового или функционального?

Нет, с версии Реакт 16 разницы нет.

Можно ли инициализировать state не внутри метода constructor?

Да. Если первоначальное состояните на зависит от передаваемых пропсов, то можно его инициализировать как обычное свойство объекта.

Что происходит, когда меняется одно или более свойств состояния state (либо одно или более свойств передаваемого объекта props)?

Вызывается повторно метод компонента render() и происходит перерисовка компонента.

Как надежно привязать метод к объекту при помощи нового синтаксиса? Как это делалось раньше в компонентах-классах.

Если значением свойства является стрелочная функция, то она навсегда привяжется к объекту, в котором была создана. Раньше методы привязывались при помощи метода функции bind в методе класса constructor.

Можно ли менять state напрямую?

Нет, только при помощи метода setState.

Что происходит, когда метод setState() принимает объект?

Он обновляет state в соответствии с этим объектом. Причем обновляются только те свойства, которые указаны в переданном объекте.

Может ли метод setState принимать функцию? Какие аргументы она принимает?

Да, может. Она принимает два рагумента prevState и prevProps - предыдущее состояние и предыдущий объект пропсов соответственно. Вернуть должна объект для обновления state.

Какая библиотека позволяет контролировать типы данных передаваемых пропсов?

Это библиотека prop-types. Ее нужно устанавливать дополнительно.

Под каким именем обычно импортируется модуль из библиотеки prop-types?

Под именем PropTypes (но это имя может быть любым).

Какое свойство используется для того, чтобы запустить проверку пропсов?

Это статическое свойство компонента propTypes, которое ссылается на объект с настройками.

Возможна ли проверка типов пропсов в функциональных компонентах?

Да.

Как можно задать проверку пропа на соответствие определенном вуди данных?

В объекте настроек в качестве ключа нужно указать имя пропа, а в качестве свойства соответствующий валидатор из библиотеки prop-types.

                        
App.propTypes = {
    count: PropTypes.number
}

Перечислите, по каким типам данных, которые есть в ES, можно валидировать пропсы?

array, bool, func, number, object, string, symbol

Что делает валидатор PropTypes.node?

Соответствует ли проп типу данных, которые могут быть отрендеренные. Это числа, строки, элементы фрагменты или массивы их содержащие.

Что делаю валидаторы arrayOf() и objectOf()? Какой аргумент они принимают?

Данные валидаторы проверяют, соответствует ли каждый элемент массива, передаваемого в пропе, или каждое свойство объекта, передаваемого в пропе, определенному типу данных из ES. Данные методы принимают валидатор нужно типа данных.

                    
App.propTypes = {
    arr: PropTypes.arrayOf(PropTypes.number),
    obj: PropTypes.objectOf(PropTypes.func)
}

Что делает валидатор PropTypes.element?

Проверяет, соответствует ли проп реакт-элементу.

Что делает валидатор PropTypes.instanceOf() и какой аргумент принимает?

Проверяет, является ли переданный проп экземпляром класса. Имя класс передается в аргументе (не строка!).

Что делает валидатор PropTypes.oneOf()?

Проверяет, соответствует ли значение пропа одному из элементов массива, который передается в качестве аргумента валидатору.

Что делает валидатор PropTypes.oneTypesOf()?

Проверяет, соответствует ли проп одному из типу данных, перечисленных в массиве, который передается в качестве аргумента валидатору. В массиве элементы соответствуют валидаторам типов.

Что делает валидатор PropTypes.shape()?

Проверяет, соответствует ли указанное свойство объекта, который передается в видео пропа, определенному типу данных.

                    
App.propTypes = {
    obj: PropTypes.shape({
        color: PropTypes.string,
        fontSize: PropTypes.number
    })
}

Можно ли при использовании библиотеки prop-types указать, что определенный проп должен быть обязательным?

Да, для этого нужно добавить свойство isRequired.

Можно ли создать свой собственный валидатор?

Да. Смотреть документацию.

При помощи какого статического свойства компонента можно добавить объект с настройками со значениями по умолчанию для пропсов?

При помощи свойства defaultProps.

Как можно добавить валидацию пропов или указания для них значений по умолчанию в функциональных компонентах?

Объекту функции нужно назначить статическое свойство propTypes или defaultProps и далее все как в классовых компонентах.

Как выглядит объект с настройками для указания значений по умолчанию для пропосв?

В нем ключ - это имя пропса, свойство - это значение по умолчанию данного пропса.

Являются ли события в React кроссбраузерными?

Да.

Для чего нужен компонент Fragment?

Он оборачивает несколько react-элементов для того, чтобы можно было их отрендерить без необходимость оборачивать одним корневым элементом.

Можно ли использовать стили инлайново?

Да. Для этого нужно передать свойству style объект с настройками css.

Чтоб импортировать в React файл со стилеми, нужны ли какие-либо дополнительные настройки.

Нет. Нужно сделать только импорт файла.
