Сколько типов данных существует в JS?

Существует 8 типов данных: 7 приметивных и объекты.

Может ли последнее свойство объекта заканчиваться запятой?

Да. Такая запятая называется "висячей запятой".

Какаво преимущество квадратных скобок перед точечной нотацией?

При помощи квадратных скобок можно обращаться к совйствам либо задавать свойства объекта, которые состоят из нескольких слов, а также обращаться через переменную.

                    
let user_name = obj["user name"];

const name = "user name";
let user_name = obj[name];

Что такое вычисляемые свойства?

Вычисляемое свойство задается при помощи квадратных скобок, в которых непосредственно записывается выражение.

                    
const prop = "name";

const obj = {[prop]:"tema",
             ['first_' + prop]: "labar",
}

Обратите внимание, что в вырожение может быть довольно сложным. Вы заметили висячую запятую?

Если значение свойства объекта берется из переменной, которая по написанию совпадает с этим свойством, как можно сократить запись создания объекта?

Этот синтаксис доступен ES6!

                    
let name = "tema";
let age = 35;

const obj = {
    age,  //age:35
    name  //name:"tema"
}

Может ли имя свойства объекта(ключ) совпадать с зарезервированными словами либо быть числом?

Да. Стоит отметить, что ключами могут быть строки либо symbol, все другие типы данных преобразуются к строке.

                    
const obj = {
    0: 'start'  //"0":"start"
}

obj["0"]

Если свойство задается при помощи числа, то доступ к нему можно получить при помощи квадратных скобок.

Что будет, если обратиться к несуществующему свойству объекта?

Будет возвращено значение underfind (а это один из 7 приметивных типов)

Отличительная особенность js в том, что при этом не будет возникать ошибка.

Для чего нужен оператор in?

При помощи данного оператора можно проверить, есть ли определенной свойство в объекте.

                    
"name" in obj;//имя

prop in obj;  //переменная

Если ищем по конкретному имени, то нужно указать в ковычках. Также можно использывать переменную.

}

При каждой итерации в переменную key записывается имя объекта. Вы обратили внимание, как при помощи квадратных скобок можно вывести значение свойства?

Как упорядочиваются свойства в объекте?

Целочисленные свойства упорядочиваются в порядке возрастания, остальные в зависимости от порядка создания.

Как можно клонировать объект?

Это можно сделать в цикле либо используя метод Object.assign()

                    
const clone = Object.assign({}, obj1, obj2);

При помощи данного метода можно объединить несколько объектов в один. Следует помнить, что таким обарзом создается поверхностная копия. Если нужно сделать глубокую копию, то нужно проверять каждое свойство при помощи рекурсии, либо использовать библиотеку lodash

Какой способ сокращенного написания методов в литерале объекта появился в ES6?

                    
const obj = {
    sayHello(){
        alert('Hello');
    }
}

//Раньше нужно было так:
const obj = {
    sayHello: function(){
        alert('Hello');
    }
}

Как можно получить доступ к объекту из метода?

Это можно сделать при помощи ключевого слова this

Значение ключевого слова this не фиксированное, он зависит от контекста, в котором вызывается метод(функция). Любая функция может иметь ключевое слово this и какое значение получит это ключевое слово, зависит от места, в котором данная функция будет вызвана. Если вызвать функцию в глобальной области выдимости, то в строгом режиме будет получено значение undefined, а в нестрогом - глобальный объект window.

На что ссылается ключевое слово this в стрелочной функции?

В стрелочной функции ключевое слово this не имеет своего значения. Она берет его из внешней "нормальной" функции.

Если стрелочная функция не вложена в "нормальный" метод объекта, а сама явялется методом объекта, то this в таком случае будет ссылаться глобальный объект в нестрогом режиме или undefind при строгом.

Какие основные два отличия функции-конструктора от обычной функции?

1. Имя функции-конструктора начинается с большой буквы.

2. Функция конструктор вызывается при помощи ключевого слова new.

Что делает ключевое слово new при вызове функции?

Неявно создает новый пустой объект

Присваевает новый объект ключевому слову this из функции.

В результате испольнения кода возвращает this (то есть созданный объект)

                    
let obj = new funciton(){};  //obj === {}

Это можно сделать в цикле либо используя метод Object.assign()

                    
const clone = Object.assign({}, obj1, obj2);

При помощи данного метода можно объединить несколько объектов в один. Следует помнить, что таким обарзом создается поверхностная копия. Если нужно сделать глубокую копию, то нужно проверять каждое свойство при помощи рекурсии, либо использовать библиотеку lodash

Как работает return в функции-конструкторе?

Функция конструктор всегда возвращает this(объект) по умолчанию и указывать данное ключевое слово не нужно. Однако, если мы укажем, чтобы функция вернула другой объект, то будет возвращен именно он. Во всех других случаях будет возвращен this. несомтря на то, чтобы мы не прописали для возврата.

                    
return 5+5  //все равно вернет this

Можно ли вызвать функцию-конструктор без скобок?

Да, если мы не передаем в нее аргументы. Однако, это считается плохой практикой.

Для чего нужно "опциональная цепочка "?."?

Опциональная цепочка ?. — это безопасный способ доступа к свойствам вложенных объектов, даже если какое-либо из промежуточных свойств не существует. Эта возможность была добавлена в язык недавно, поэтому для старых версий браузеров потребуется полифил.

                    
let value = user?.adderss?.street?.home?.room

user?.actions?.sayHello?.();  //проверит, явяляется ли sayHello методом, и если да, то вызывает его. 
let value = user?.adderss?.['user street'];

Опчиональная цепочка ?. проверяет, явялется ли выражение слева от нее объектом. Елси да, то передает значение этого выражения дальще по цепочке, если же нет, то возвращает undefind. Ее можно исопльзовать с квадрантыми и круглыми скобками, чтобы получать свойство объекта или вызывать метод.

Какие два типа данных могут быть использованы в качестве ключей свойств объектов?

Это могут быть либо строки, либо символы.

Что такое символ и как его можно создать?

Символ это уникальный индетификатор. Создается он при помощи функции Symbol()

                    
let id = Symbol();

При создании символа ему можно передать описание, но оно ни на что не влияет и используется больше для отладки. Символы гарантировано уникальны. Один символ никогда не равен другом, даже если имеют одинаковое описание.

Символы неявно не преобразуются в строки, это можно сделать только явно при помощи метода .toString()/. Также символы имеют специальное свойство description, которое возвращает описание.

                    
let id = Symbol('unicum id');
id.description  // unicum id

Для чего нужны символы в объектах?

Поскольку символы всегда уникальны, при использовании сторонней библиотеки и необходимости добавить в объекты свои свойства, это предохраняет нас от того, что мы сможем перезатерт первоначальные свойства объекта.

                    
let id = Symbol();
let obj = {[id]: "number id"}
alert(obj[id]);

Обратите внимание, как в объектном лирерале объявляется вычисляемое свойство при помощи []. В цикле for...in свойства, которые имеют в качестве ключа символ, не перечилсяются. При клонировании при помощи функции Object.assign()такие свойства копируются в новый объект.

Что такое глобальные символы?

Это символы, которые содержаться в глобальном реестре. Они нужны для того, чтобы иметь к ним доступ из любого места приложения. Глобальные символы с одинаковым описанием равны между собой.

                    
let id = Symbo.for(id);

При помощи Symbo.for() можно получить символ из глобального реестра, если он там есть. Если же его там нет, то такой символ будет создан. Глобальные символы доступны везде в коде.

Как можно получить имя символа?

                    
const id = Symbol.key('name for global symbol');
let nameSymbol = Symbol.keyFor(id);  //name for global symbol

Стоит помнить, что данный метод работает только для глобальных символов. Для обычных, как мы помним, есть свойство discription.

Существует множество системных символов, которые используются внутри JS. Их можно использовать для настройки различных аспектов поведения объектов. Например для настройки правила, по которому объект будет преобразовываться к примитиву.

Как объект преобразуется к примитивному типу?

Все зависит от того, в каком контексте производится преобразование, а именно, что нужно получить: логическое значение, стоку или число. Если мы выполняем матиматическую опирацию, то преобразовать нужно к число(хинт number), если логическую, то к логическому значению (хинт boolean)

Для объектов можно выделить три хинта: "string", "number", "default".

В процессе приобразования к примитиву, движок пытается вызвать следующие методы:

ojb[Symbol.toPrimitive](hint) - если такой метод существует, и передает ему хинт. Обратите внимание, что ключом является символ (в данном случае системный символ)

hint === 'string' - сначало вызывается .toString(), если такого нет, то .valueOf()

hint === 'number' || 'default' - сначало вызывается .valueOf(), если такого нет, то .toString()

                    
obj[Symbol.toPrimitive] = function(hint){
    if(hint === "string"){ return "String"}
    if(hint === "number" || hint === "default"){ return "Number"}
}

Методы должны возвращать примитивный тип, а не объект. Методы toString() и valueOf() это устаревшие способы преобразований и были реализованы тогда, когда еще не было символов. На практике довольно часто достаточно реализовать только obj.toString() как «универсальный» метод для всех типов преобразований, возвращающий «читаемое» представление объекта, достаточное для логирования или отладки
