На что ссылается скрытое свойство объекта [[Prototype]]?

Оно либо равно null, либо ссылается на другой объект. Этот объект называется «прототип»

Можно ли при помощи свойства __proto__ получить либо установить прототип объекта?

Да. Оно считается устаревшим, но по факту продолжает работать во многих средах. Данное свойство является геттером/сеттером.

Перечисляет ли цикл for … in свойства и методы из объекта-прототипа?

Да

Что делает obj.hasOwnProperty(key)?

Метод проверяет, если собственное ненаследуемое свойство в объекте obj с ключом key. Возвращает true либо false.

На что ссылается свойство Function.prototype?

На объект, который будет установлен в качестве прототипа нового созданного объекта. Этот свойство используется только в момент вызова конструктора и создания объекта.

По умолчанию свойство Function.prototype ссылается на объект с одним свойством. Какое это свойство?

{constructor: Function} - это свойство ссылается на функцию-конструктор.

Почему при создании объекта при помощи литерала {}, у него есть методы, как toString(), valueOf() и так далее? Ведь мы их не объявляли.

При создании объекта при помощи литерала неявно вызывается конструктор new Object() и новый объект получает прототип Object.prototype. Из него и берутся методы и свойства, которые мы сами не объявляли.

Какой объект-прототип у массива? Какой объект-прототип, в свою очередь, у этого объекта?

У массива прототипом является Array.prototype, который в свою очередь имеет прототип Object.prototype
[] => Array.prototype => Object.prototype => null

Что будет, если изменить прототип встроенного объекта? Получат ли уже созданные объекты эти изменения?

Все уже созданные объекты получать изменения. Этим встроенные объекты отличаются от тех, которые мы создаем при помощи своих конструкторов.

В каких случаях считается оправданным изменения прототипов встроенных объектов?

При создании полифилов.

Можно ли взять метод из прототипа и присвоить его свойству другого объекта?

Да.

                    
obj.join = Array.prototype.join;
obj.join(': ');

Что делает метод Object.create(prototype, [descriptors])?

Создает новый объект и устанавливает ему прототип prototype. Вторый необязательным параметром принимает объект дискриптор.

Что делает метод Object.getPrototypeOf(obj)?

Позволяет получить прототип объекта.

Что делает метод Object.setPrototypeOf(obj, prototype)?

Устанавливает прототип объекта.

Что делают методы Object.keys(obj) / Object.values(obj) / Object.entries(obj)?

Методы позволяют получить массив перечисляемых собственных ключей / массив перечисляемых собственных значений / массив перечисляемых собственных пар [key, value].

Что делает методы Object.getOwnPropertySymbols(obj)?

Возвращает массив всех символьных ключей объекта.

Что делает методы Object.getOwnPropertyNames(obj)?

Возвращает массив всех строковых ключей объекта.

Что делает методы Reflect.ownKeys(obj)?

Возвращает массив всех ключей объекта.
