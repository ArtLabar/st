Как можно инициализировать репозиторий?

Это можно сделать при помощи команды:

                    
git init

Эта команда создаст в текущей директории поддиректорию .git, которая содержит структурные файлы.

Как можно инициализировать репозиторий?

Это можно сделать при помощи команды:

                    
                        git add .

При помощи какой команды можно создать коммит?

Это можно сделать при помощи команды:

                    
                        git commit -m "first commit"

Комментарий пишется в ковычках.

Как можно клонировать удаленный репозиторий?

Это можно сделать при помощи команды:

                    
                        git clone https://github.com/libgit2/libgit2

При этом копируется не только рабочая версия проекта, а все репозиторий полностью. В этом отличие git от многих другий систем. Если нужно задать свое имя директорию, то его нужно указать в данной команде после url (указывается без ковычек)

При помощи какой команды можно узнать статус?

Это можно сделать при помощи команды:

                    
                        git status

Эта команда покажет, имеются ли измененные, неотслеживаемые и неиндексируемые файлы.

                    
                        git status -s

Так можно посмотреть статус в более коротком виде.

Как можно сделать, чтобы определенные файлы не отображались в списке неотслеживаемых файлов?

Для этих целей существует специальный файл .gitignore. Указываются данные файлы при помощи glob-шаблоно (упрощенной версии регулярных выражений).

При помощи какой команды можно посмотерть конкретно поменялось, а не только какие файлы были изменены?

При помощи команды:

                    
                        git diff

Эта команда позволяет получить ответ на два вопроса: что вы изменили, но ещё не проиндексировали, и что вы проиндексировали и собираетесь включить в коммит. Важно помнить, что вызов команды без аргументов позволит увидить изменения которые не проиндексированы. Важно отметить, что git diff сама по себе не показывает все изменения сделанные с последнего коммита — только те, что ещё не проиндексированы. Если вы хотите посмотреть, что вы проиндексировали и что войдёт в следующий коммит, вы можете выполнить git diff --staged.

Чем отличается команда git commit от команды git commit -m?

Команда git commit откроет выбранный вами текстовый редактор, при помощи которого можно прописать комментарии к коммиту.

При помощи какйо одной команды можно одновременно проиндексировать все измененные файлы и создать новый коммит?

Это можно сделать при помощи команды:

                    
                        git commit -a -m "information"

Как нужно удалять файлы в репозитории?

Это можно сделать при помощи команды:

                    
                        git rm

Если вы просто удалите файл из своего рабочего каталога, он будет показан в секции “Changes not staged for commit” (измененные, но не проиндексированные) вывода команды git status. Если вы выполните команду git rm, удаление файла попадёт в индекс. После следующего коммита файл исчезнет и больше не будет отслеживаться. Если вы изменили файл и уже проиндексировали его, вы должны использовать принудительное удаление с помощью параметра -f.

При помощи какой команды можно удалить файл из индекса, оставив его при этом в рабочем каталоге?

Это можно сделать при помощи команды:

                    
                        git rm --cached

В команду git rm можно передавать файлы, каталоги или шаблоны.

При помощи какой команды можно переименовать либо переместить файл в git?

Это можно сделать при помощи команды:

                    
                        git mv

При помощи данной команды можно как переименовать файл, так и переместить его. Для перемещения нужно прописывать полный путь первоначального местоположения файла на полный путь места, куда нужно его переместить:

                    
                        git mv 'old place/file.txt' 'new place/file.txt'

Как можно посмотреть историю коммитов?

Это можно сделать при помощи команды:

                    
                        git log

Команда git log имеет очень большое количество опций для поиска коммитов по разным критериям.

Как можно инициализировать репозиторий?

Какой аргумент команды git log показывает изменения, внесенные в каждый коммит? Как можно ограничить число выводимых коммитов?

                    
                        git log -p -4

-p это то же самое, что и --pathc. Для того, чтобы посмотреть сокращенную статистику, нужно использывать опцию --stat

                    
                        git log --stat

Расскажите, для чего нужна опция --pretty? Как ею пользываться?

Эта опция меняет формат вывода. Существует несколько встроенных вариантов отображения. Опция oneline выводит каждый коммит в одну строку, что может быть очень удобным если вы просматриваете большое количество коммитов. К тому же, опции short, full и fuller делают вывод приблизительно в том же формате, но с меньшим или большим количеством информации соответственно

                    
                        git --pretty=online

Наиболее интересной опцией является format, которая позволяет указать формат для вывода информации.

                    
                        git log --pretty=format:"%h - %an, %ar : %s"

Для чего нужно опция --graph? Как ею пользываться?

С этой опцией вы сможете увидеть небольшой граф в формате ASCII, который показывает текущую ветку и историю слияний:

                    
                        git log --graph

Что делают опции --since и --until команды git log?

Это опции для ограничения вывода по времени.

                    
                        git log --since=2.weeks

Это команда работает с большим количеством форматов — вы можете указать определенную дату вида "2008-01-15" или же относительную дату, например "2 years 1 day 3 minutes ago".

Как найти коммиты по имени автора либо по ключевому слову в комментарии к коммиту?

Это можно сделать при помощи опций --author и --grep соответственно:

                    
                        git log --author=gitster

Что делает опция -S?

Данная опция принимает аргумент в виде строки и показывает только те коммиты, в которых изменение в коде повлекло за собой добавление или удаление этой строки. Например, если вы хотите найти последний коммит, который добавил или удалил вызов определенной функции, вы можете запустить команду:

                    
                        git log -S function_name
